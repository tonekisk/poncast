# PonCast (v0.5 リリース) 📺

![Python](https://img.shields.io/badge/Python-3.8%2B-blue?logo=python) ![License](https://img.shields.io/badge/license-MIT-green) ![Platform](https://img.shields.io/badge/platform-macOS-lightgrey)

OBS Studio と連携するポッドキャスト・配信向け **Webテロップ制御システム**。
ブラウザ上のコントロールパネルからテロップ・画像・動画・スクロールテキストをリアルタイムで操作し、WebSocket 経由で OBS のブラウザソースに即時反映します。

## 最新アップデート (v0.5) - 2026/03/24 🚀

- **macOS メネバーアプリ化**: ターミナルを表示せず、メニューバーからサーバー状態の監視・制御が可能になりました。
- **インテリジェント起動**: アプリを開くと、最適な Python を自動探索し、ダッシュボードを即座に開きます。

---

## 起動方法

### macOS (推奨)
1. **Mac.PonCast.app** をダブルクリックします。
2. メニューバーにアイコンが表示され、自動的にサーバーが立ち上がりダッシュボードが開きます。
3. 停止する場合は、メニューバーのアイコンから「Quit」を選択してください。

### Windows
1. **Win.start.bat** をダブルクリックします。
2. ターミナルが開き、サーバーが開始されます。

## 主な機能

| 機能 | 説明 |
|---|---|
| **テロップ表示** | ロウワーサード形式のテキストオーバーレイ |
| **メディア表示** | 画像・動画のオーバーレイ（アップロード対応） |
| **ブレイク画面** | 休憩・ブレイク画面の切り替え |
| **スクロールテキスト** | 横スクロールするテキスト表示 |
| **シーン管理** | 場面（タブ）単位でコンテンツを整理 |
| **一斉オフ** | 全コンテンツをワンクリックで非表示 |
| **リアルタイム反映** | WebSocket によりコントロールパネルの操作が OBS に即時反映 |

## セットアップ（初回のみ）

Python の `websockets` ライブラリが必要です。ターミナルまたはコマンドプロンプトで以下を実行してください。

```bash
pip3 install websockets
```

---

## OBS の設定

1. OBS Studio を開く
2. 「ソース」→「+」→「ブラウザ」を追加
3. URL に `http://localhost:5500/obs_display.html` を入力
4. 幅・高さを配信解像度（例: 1920 × 1080）に合わせる
5. 「シーンに合わせる」でシーン全体をカバー

> **注意:** ブラウザソースの「シャットダウン（非表示時）」は**オフ**にすると接続が維持されます。

### サーバー構成

| ポート | 役割 |
|---|---|
| 5500 | HTTP サーバー（静的ファイル配信 + ファイルアップロード） |
| 5501 | WebSocket サーバー（クライアント間メッセージリレー） |

## ファイル構成

```
PonCast/
├── server.py              # HTTPサーバー + WebSocketリレーサーバー
├── start.command          # Mac用起動スクリプト（ダブルクリックで起動）
├── requirements.txt       # Python依存ライブラリ
├── .gitignore
├── README.md
└── public/
    ├── PonCast.html       # メインコントロールパネル
    ├── obs_display.html   # OBSブラウザソース用表示レイヤー
    └── uploads/           # アップロードされたメディアファイル（.gitignore対象）
```

## 停止方法

- **`start.command` で起動した場合:** ターミナルウィンドウで `Ctrl+C`
- **手動起動の場合:** `Ctrl+C`
